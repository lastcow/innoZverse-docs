# Lab 2: Filesystem Hierarchy

## 🎯 Objective
Explore the Linux Filesystem Hierarchy Standard (FHS) and understand the purpose of key directories like `/etc`, `/var`, `/home`, `/usr`, and `/tmp`.

## ⏱️ Estimated Time
25 minutes

## 📋 Prerequisites
- Completed Lab 1 (Terminal Basics)
- Basic comfort with the terminal prompt

## 🔬 Lab Instructions

### Step 1: View the Root of the Filesystem
The Linux filesystem starts at `/` (root). Everything lives under it.

```bash
ls /
# Output:
# bin   dev  home  lib    lib64  lost+found  mnt  opt   proc  run   srv  tmp  var
# boot  etc  init  lib32  libx32  media       mntp  proc  root  sbin  sys  usr
```

### Step 2: Explore `/etc` — Configuration Files
`/etc` contains system-wide configuration files. No binaries live here.

```bash
ls /etc | head -20
# Shows the first 20 entries

# View hostname config
cat /etc/hostname

# View OS release info
cat /etc/os-release

# View timezone setting
cat /etc/timezone
```

### Step 3: Explore `/var` — Variable Data
`/var` holds data that changes frequently: logs, mail, databases, package lists.

```bash
ls /var
# Output: backups  cache  lib  local  lock  log  mail  opt  run  spool  tmp

# View system logs
ls /var/log

# Check the size of /var/log
du -sh /var/log
```

### Step 4: Explore `/home` — User Home Directories
Each regular user gets a subdirectory in `/home`.

```bash
ls /home
# Output: student  (or your username)

# Your home directory contains personal files and config
ls -la /home/$USER
```

### Step 5: Explore `/usr` — User System Resources
`/usr` contains the majority of user-land programs and libraries.

```bash
ls /usr
# Output: bin  games  include  lib  lib32  lib64  libexec  libx32  local  sbin  share  src

# Where most commands live
ls /usr/bin | head -20

# Local programs installed manually
ls /usr/local/bin
```

### Step 6: Explore `/tmp` — Temporary Files
`/tmp` is writable by all users. Contents are usually cleared on reboot.

```bash
ls /tmp

# Create a test file
touch /tmp/mytest.txt
ls /tmp/mytest.txt
# Output: /tmp/mytest.txt

# Note: do not store important data in /tmp
```

### Step 7: Explore `/bin` and `/sbin`
On modern Ubuntu, `/bin` and `/sbin` are symlinks to `/usr/bin` and `/usr/sbin`.

```bash
ls -la /bin
# Output: lrwxrwxrwx 1 root root 7 ... /bin -> usr/bin

ls -la /sbin
# Output: lrwxrwxrwx 1 root root 8 ... /sbin -> usr/sbin
```

### Step 8: Explore `/proc` — Kernel and Process Info
`/proc` is a virtual filesystem — it doesn't exist on disk, it's generated by the kernel.

```bash
# View CPU info
cat /proc/cpuinfo | head -20

# View memory info
cat /proc/meminfo | head -10

# View kernel version
cat /proc/version
```

### Step 9: Explore `/dev` — Device Files
`/dev` contains device files representing hardware and virtual devices.

```bash
ls /dev | head -20

# Your hard disk
ls /dev/sd* 2>/dev/null || ls /dev/vd* 2>/dev/null || ls /dev/nvme* 2>/dev/null

# Terminal devices
ls /dev/tty*
```

### Step 10: Explore `/boot` — Bootloader Files
```bash
ls /boot
# Contains: vmlinuz (kernel), initrd (initial ramdisk), grub config

ls /boot/grub/
```

### Step 11: Explore `/opt` and `/srv`
```bash
# /opt: optional/third-party software
ls /opt

# /srv: service data (web, ftp)
ls /srv
```

### Step 12: Use `tree` to Visualize Hierarchy
```bash
# Install tree if not present
sudo apt install tree -y

# Show top-level directory structure
tree / -L 1

# Show /etc structure 2 levels deep
tree /etc -L 2 | head -40
```

## ✅ Verification
```bash
# Confirm key directories exist
for dir in /etc /var /home /usr /tmp /proc /dev /boot; do
  echo "$dir: $(ls $dir | wc -l) items"
done
# Each should show a non-zero number of items

# Check your home dir path
echo $HOME
# Output: /home/yourusername
```

## 📝 Summary
- `/` is the root of everything — no drive letters in Linux
- `/etc` = configuration; `/var` = variable data/logs; `/home` = user files
- `/usr` = programs and libraries; `/tmp` = temporary (wiped on reboot)
- `/proc` and `/dev` are virtual filesystems — not real files on disk
- Understanding the FHS helps you know where to look when troubleshooting
