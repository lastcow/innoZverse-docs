## Step 1: Network Interfaces

```bash
ip addr show
```

**📸 Verified Output:**
```
1: lo: <LOOPBACK,UP,LOWER_UP> mtu 65536 qdisc noqueue state UNKNOWN group default qlen 1000
    link/loopback 00:00:00:00:00:00 brd 00:00:00:00:00:00
    inet 127.0.0.1/8 scope host lo
       valid_lft forever preferred_lft forever
2: eth0: <BROADCAST,MULTICAST,UP,LOWER_UP> mtu 1500 qdisc fq_codel state UP group default qlen 1000
    link/ether 02:42:ac:11:00:06 brd ff:ff:ff:ff:ff:ff
    inet 172.17.0.6/16 brd 172.17.255.255 scope global eth0
       valid_lft forever preferred_lft forever
```

Key fields:
- **lo**: Loopback interface — `127.0.0.1`, traffic stays on this machine
- **eth0**: Network interface — `172.17.0.6/16` is the container's IP
- `/16` = subnet mask (covers `172.17.0.0` to `172.17.255.255`)

> 💡 Every Linux server has at least `lo` (loopback). Real servers have `eth0`, `ens3`, `enp0s3`, etc. Cloud VMs may have `ens3` or `ens160`. The name depends on the driver and BIOS.

## Step 3: DNS Configuration

```bash
cat /etc/resolv.conf
```

**📸 Verified Output:**
```
# Generated by Docker Engine.
nameserver 8.8.8.8
search .
```

> 💡 `nameserver 8.8.8.8` = Google's DNS. When you type `curl google.com`, the system asks `8.8.8.8` to resolve `google.com` to an IP address. Without a nameserver, hostnames won't resolve.

```bash
# Manual DNS lookup (using getent, available without extra packages)
getent hosts localhost
```

**📸 Verified Output:**
```
127.0.0.1       localhost
```

## Step 4: Routing Table

```bash
ip route show
```

**📸 Verified Output:**
```
default via 172.17.0.1 dev eth0
172.17.0.0/16 dev eth0 proto kernel scope link src 172.17.0.6
```

- **default via 172.17.0.1** — all traffic not matching other routes goes to this gateway
- **172.17.0.0/16** — this subnet is directly accessible via eth0

> 💡 The routing table is your network map. Traffic to 172.17.x.x goes directly. Everything else hits the default gateway. Misconfigured routing = no network access, even if the interface is up.

## Step 5: Checking Open Ports with ss

```bash
ss -tlnp
```

**📸 Verified Output:**
```
Netid  State   Recv-Q Send-Q Local Address:Port  Peer Address:Port
```

```bash
# All sockets (including UDP)
ss -alnp | head -10
```

**📸 Verified Output:**
```
Netid  State    Recv-Q Send-Q Local Address:Port   Peer Address:Port  Process
nl     UNCONN   0      0       0                    0
nl     UNCONN   0      0       0                    0
```

> 💡 `ss` replaced `netstat` on modern Linux. `-t`=TCP, `-u`=UDP, `-l`=listening, `-n`=numeric (no DNS lookups), `-p`=show process. On a real server: `ss -tlnp` shows you every open port and which process owns it — essential for security audits.

## Step 7: ping — Connectivity Test

```bash
ping -c 3 127.0.0.1
```

**📸 Verified Output:**
```
PING 127.0.0.1 (127.0.0.1) 56(84) bytes of data.
64 bytes from 127.0.0.1: icmp_seq=1 ttl=64 time=0.038 ms
64 bytes from 127.0.0.1: icmp_seq=2 ttl=64 time=0.040 ms
64 bytes from 127.0.0.1: icmp_seq=3 ttl=64 time=0.041 ms

--- 127.0.0.1 ping statistics ---
3 packets transmitted, 3 received, 0% packet loss, time 2047ms
rtt min/avg/max/mdev = 0.038/0.039/0.041/0.001 ms
```

> 💡 `ping` sends ICMP echo requests. **0% packet loss** = connectivity OK. High RTT = latency. Some hosts block ICMP (firewalls), so ping failure ≠ host is down — always verify with `curl` too.

## Step 8: Capstone — Network Diagnostic Script

```bash
cat > /tmp/netdiag.sh << 'SCRIPT'
#!/bin/bash
echo "================================================"
echo "  Network Diagnostic Report"
echo "  $(date)"
echo "================================================"
echo ""

echo "1. HOSTNAME & IP"
echo "   Hostname: $(hostname)"
echo "   IP:       $(hostname -i 2>/dev/null || ip addr show eth0 2>/dev/null | grep 'inet ' | awk '{print $2}')"
echo ""

echo "2. INTERFACES"
ip addr show | grep -E 'inet |^[0-9]' | while read line; do
    echo "   $line"
done
echo ""

echo "3. DEFAULT GATEWAY"
GW=$(ip route | grep default | awk '{print $3}')
echo "   Gateway: ${GW:-not found}"
echo ""

echo "4. DNS"
NS=$(grep nameserver /etc/resolv.conf | awk '{print $2}')
echo "   Nameserver: ${NS:-not configured}"
echo ""

echo "5. LOOPBACK TEST"
if ping -c 1 -W 1 127.0.0.1 > /dev/null 2>&1; then
    echo "   ✅ Loopback (127.0.0.1): OK"
else
    echo "   ❌ Loopback: FAILED"
fi
echo ""

echo "6. OPEN PORTS"
PORTS=$(ss -tlnp 2>/dev/null | grep -c LISTEN)
echo "   Listening TCP ports: $PORTS"
ss -tlnp 2>/dev/null | grep LISTEN | awk '{print "   " $4}' | head -5
echo ""

echo "7. INTERNET CONNECTIVITY"
if curl -sI --max-time 3 http://example.com > /dev/null 2>&1; then
    echo "   ✅ HTTP (example.com): reachable"
else
    echo "   ⚠️  HTTP (example.com): unreachable (may be expected in container)"
fi
echo ""

echo "================================================"
echo "  Diagnostic complete"
echo "================================================"
SCRIPT

chmod +x /tmp/netdiag.sh
/tmp/netdiag.sh
```

```

   1: lo: <LOOPBACK,UP,LOWER_UP> mtu 65536 ...

   2: eth0: <BROADCAST,MULTICAST,UP,LOWER_UP> ...

================================================
```

## Summary

| Command | Purpose |
|---------|---------|
| `ip addr show` | List network interfaces and IPs |
| `ip route show` | Routing table |
| `hostname` | Show hostname |
| `hostname -i` | Show primary IP |
| `cat /etc/hosts` | Local DNS overrides |
| `cat /etc/resolv.conf` | DNS server config |
| `ss -tlnp` | TCP listening ports |
| `ping -c N host` | ICMP connectivity test |
| `curl -sI url` | HTTP header check |
| `curl -sL url -o file` | Download file |
| `getent hosts name` | DNS lookup |
