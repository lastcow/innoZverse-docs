# Lab 09: DNS Fundamentals

**Time:** 30 minutes | **Level:** Foundations | **Docker:** `docker run -it --rm ubuntu:22.04 bash`

---

## Overview

The **Domain Name System (DNS)** is the internet's phone book — it translates human-readable names like `google.com` into IP addresses like `142.251.35.110`. Without DNS, you'd need to memorize IP addresses for every website. This lab explores the DNS hierarchy, record types, resolution process, and essential DNS tools.

---

## Step 1: Install DNS Tools & Run First Query

```bash
docker run --rm ubuntu:22.04 bash -c "
apt-get update -qq 2>/dev/null &&
apt-get install -y -qq dnsutils 2>/dev/null &&
echo '=== dig google.com A ===' &&
dig google.com A +short &&
echo '' &&
echo '=== cat /etc/resolv.conf ===' &&
cat /etc/resolv.conf
"
```

📸 **Verified Output:**
```
=== dig google.com A ===
142.251.35.110

=== cat /etc/resolv.conf ===
# Generated by Docker Engine.
# This file can be edited; Docker Engine will not make further changes once it
# has been modified.

nameserver 8.8.8.8
search .

# Based on host file: '/run/systemd/resolve/resolv.conf' (legacy)
# Overrides: []
```

> 💡 **Tip:** `/etc/resolv.conf` tells your system which DNS server to use. `nameserver 8.8.8.8` means all DNS queries go to Google's public resolver. The `search` directive appends domain suffixes when resolving short hostnames.

## Step 3: DNS Record Types

```bash
docker run --rm ubuntu:22.04 bash -c "
apt-get update -qq 2>/dev/null &&
apt-get install -y -qq dnsutils 2>/dev/null &&
echo '=== A record (IPv4) ===' &&
dig google.com A +short &&
echo '=== AAAA record (IPv6) ===' &&
dig google.com AAAA +short &&
echo '=== MX record (mail) ===' &&
dig google.com MX +short &&
echo '=== NS record (name servers) ===' &&
dig google.com NS +short &&
echo '=== TXT record ===' &&
dig google.com TXT +short | head -3
"
```

📸 **Verified Output:**
```
=== A record (IPv4) ===
142.251.35.110

=== AAAA record (IPv6) ===
2607:f8b0:4008:80c::200e

=== MX record (mail) ===
10 smtp.google.com.

=== NS record (name servers) ===
ns1.google.com.
ns2.google.com.
ns3.google.com.
ns4.google.com.

=== TXT record ===
"v=spf1 include:_spf.google.com ~all"
"apple-domain-verification=30afIBcvSuDV2PLX"
"globalsign-smime-dv=CDYX+XFHUw2wml6/Gb8+59BsH31KzUr6c1l2BfMLpBGqh/x9"
```

> 💡 **Tip:** MX records include a **priority number** (lower = higher priority). `10 smtp.google.com` means all mail goes to smtp.google.com at priority 10. Multiple MX records = failover.

## Step 6: Recursive vs Iterative Resolution

```bash
docker run --rm ubuntu:22.04 bash -c "
python3 -c \"
print('DNS Resolution Types:')
print()
print('RECURSIVE (client asks resolver to do all work):')
print('  Client → Resolver: \"What is the IP for www.example.com?\"')
print('  Resolver → Root: \"Who has .com?\"')
print('  Root → Resolver: \"Ask a.gtld-servers.net\"')
print('  Resolver → TLD: \"Who has example.com?\"')
print('  TLD → Resolver: \"Ask ns1.example.com\"')
print('  Resolver → Auth: \"What is www.example.com?\"')
print('  Auth → Resolver: \"93.184.216.34\"')
print('  Resolver → Client: \"93.184.216.34\" (and caches it!)')
print()
print('ITERATIVE (client does the work itself — rare):')
print('  Client → Root: \"www.example.com?\"')
print('  Root → Client: \"Ask TLD at a.gtld-servers.net\"')
print('  Client → TLD: \"www.example.com?\"')
print('  TLD → Client: \"Ask authoritative at ns1.example.com\"')
print('  Client → Auth: \"www.example.com?\"')
print('  Auth → Client: \"93.184.216.34\"')
print()
print('Caching:')
print('  Each answer has a TTL (Time To Live) in seconds')
print('  Resolver caches the answer for TTL seconds')
print('  Next query is answered instantly from cache (no round trip!)')
print()
print('  google.com A TTL=300  → cached for 5 minutes')
print('  cloudflare.com A TTL=60 → cached for 1 minute')
print('  large CDNs use low TTL for fast failover')
\"
"
```

📸 **Verified Output:**
```
DNS Resolution Types:

RECURSIVE (client asks resolver to do all work):
  Client → Resolver: "What is the IP for www.example.com?"
  Resolver → Root: "Who has .com?"
  Root → Resolver: "Ask a.gtld-servers.net"
```

## Step 7: /etc/hosts, Split-Horizon DNS & DNS over HTTPS

```bash
docker run --rm ubuntu:22.04 bash -c "
echo '=== /etc/hosts content ===' &&
cat /etc/hosts &&
echo '' &&
python3 -c \"
print('DNS Resolution Order (/etc/nsswitch.conf):')
print('  1. /etc/hosts  (local file — checked FIRST)')
print('  2. DNS server  (/etc/resolv.conf)')
print('  3. mDNS        (.local domains on LAN)')
print()
print('/etc/hosts use cases:')
print('  127.0.0.1  localhost         (always present)')
print('  192.168.1.10  myserver.local (LAN shortcut)')
print('  0.0.0.0  ads.tracker.com     (block ads/malware!)')
print()
print('Split-Horizon DNS:')
print('  Same domain name, different answers depending on who asks:')
print('  Inside corporate network:  internal.company.com → 10.0.0.100')
print('  From the internet:         internal.company.com → NXDOMAIN (or public IP)')
print('  Used for: VPN, security, internal services')
print()
print('DNS over HTTPS (DoH):')
print('  Traditional DNS: UDP port 53, unencrypted, ISP can see/modify')
print('  DoH: HTTPS port 443, encrypted, looks like normal web traffic')
print('  Providers: Cloudflare (1.1.1.1), Google (8.8.8.8), NextDNS')
print('  URL example: https://cloudflare-dns.com/dns-query?name=google.com&type=A')
print()
print('DNS over TLS (DoT):')
print('  Like DoH but dedicated port 853, not hidden in HTTPS')
\"
"
```

📸 **Verified Output:**
```

ff00::0	ip6-mcastprefix
ff02::1	ip6-allnodes

DNS Resolution Order (/etc/nsswitch.conf):
  1. /etc/hosts  (local file — checked FIRST)
  2. DNS server  (/etc/resolv.conf)
  3. mDNS        (.local domains on LAN)

/etc/hosts use cases:
  127.0.0.1  localhost         (always present)
  192.168.1.10  myserver.local (LAN shortcut)
  0.0.0.0  ads.tracker.com     (block ads/malware!)

Split-Horizon DNS:
  Same domain name, different answers depending on who asks:
  Inside corporate network:  internal.company.com → 10.0.0.100
  From the internet:         internal.company.com → NXDOMAIN (or public IP)
  Used for: VPN, security, internal services

DNS over HTTPS (DoH):
  Traditional DNS: UDP port 53, unencrypted, ISP can see/modify
  DoH: HTTPS port 443, encrypted, looks like normal web traffic
  Providers: Cloudflare (1.1.1.1), Google (8.8.8.8), NextDNS
  URL example: https://cloudflare-dns.com/dns-query?name=google.com&type=A

DNS over TLS (DoT):
  Like DoH but dedicated port 853, not hidden in HTTPS
```

> 💡 **Tip:** DNS uses **UDP port 53** by default (fast, connectionless). For large responses (>512 bytes, like DNSSEC), it falls back to **TCP port 53**. DoH uses TCP 443 and DoT uses TCP 853.

## Step 8: Capstone — Full DNS Investigation

```bash

for rtype in A AAAA MX NS TXT; do
  echo \"--- \$rtype ---\"

done &&
echo '' &&
python3 -c \"
import subprocess, re

def dns_query(domain, rtype):
    result = subprocess.run(['dig', domain, rtype, '+short'], capture_output=True, text=True)
    return result.stdout.strip()

domains = ['google.com', 'github.com', 'cloudflare.com']
print('DNS A Record Comparison:')
print(f'  {\"Domain\":<20} {\"A Record\"}')
print('  ' + '-' * 50)
for d in domains:
    answer = dns_query(d, 'A')
    first = answer.split()[0] if answer else 'NXDOMAIN'
    print(f'  {d:<20} {first}')

print()
print('DNS query stats:')
import time
start = time.time()
r = subprocess.run(['dig', 'google.com', 'A', '+stats'], capture_output=True, text=True)
elapsed = time.time() - start
for line in r.stdout.splitlines():
```

## Summary

| Concept | Key Points |
|---|---|
| **DNS purpose** | Translates hostnames → IP addresses (and vice versa) |
| **Hierarchy** | Root → TLD → Authoritative → Recursive resolver → Client |
| **A record** | Hostname → IPv4; `dig example.com A` |
| **AAAA record** | Hostname → IPv6 |
| **CNAME** | Alias pointing to another hostname |
| **MX** | Mail server; lower priority number = higher priority |
| **PTR** | Reverse DNS; IP → hostname; uses `.in-addr.arpa` |
| **TXT** | Text data; used for SPF, DKIM, site verification |
| **TTL** | Cache lifetime in seconds; low TTL = faster changes propagate |
| **Recursive resolution** | Resolver does all the work; caches results |
| **/etc/hosts** | Local override; checked before DNS queries |
| **Split-horizon** | Different answers based on requester's network |
| **DoH** | DNS over HTTPS (port 443); encrypted; private |
| **dig** | `dig domain type +short`; best DNS debugging tool |
