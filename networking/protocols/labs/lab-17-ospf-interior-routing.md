# Lab 17: OSPF — Interior Routing Protocol

**Time:** 35 minutes | **Level:** Protocols | **Docker:** `docker run -it --rm ubuntu:22.04 bash`

---

OSPF (Open Shortest Path First) is the dominant Interior Gateway Protocol (IGP) used inside enterprise networks and ISP cores. Unlike BGP's path-vector approach, OSPF uses link-state advertisements and Dijkstra's SPF algorithm to build a complete topology map.

---

## Step 1: Install FRRouting and Explore OSPF

```bash
docker run --rm ubuntu:22.04 bash -c "
apt-get update -qq && apt-get install -y -qq frr 2>/dev/null | tail -1
echo '=== FRR Installed ==='
vtysh --help 2>&1 | head -5
echo ''
echo '=== Enable OSPF daemon ==='
sed -i 's/ospfd=no/ospfd=yes/' /etc/frr/daemons
sed -i 's/zebra=no/zebra=yes/' /etc/frr/daemons
grep -E '^ospfd|^zebra' /etc/frr/daemons
"
```

> 💡 **Link-State vs Distance-Vector:** Distance-vector (RIP) shares routing tables with neighbours — "routing by rumour." Link-state (OSPF) floods the entire topology to all routers, then each runs Dijkstra's SPF independently. Link-state converges faster and scales better.

## Step 2: OSPF Areas and Router Types

```bash
docker run --rm ubuntu:22.04 bash -c "
cat << 'EOF'
=== OSPF Area Types ===

Area 0 (Backbone):
  - All OSPF areas must connect to Area 0
  - Carries Type-1, Type-2, Type-3, Type-4, Type-5 LSAs
  - Cannot be a stub area

Normal Area:
  - Accepts all LSA types including external (Type-5)
  - Full LSDB including external routes

Stub Area:
  - Blocks Type-5 (external/AS-external LSAs)
  - ABR injects default route instead
  - Reduces LSDB size for remote sites

Totally Stubby Area (Cisco/FRR extension):
  - Blocks Type-3, Type-4, AND Type-5 LSAs
  - Only default route from ABR
  - Smallest LSDB — ideal for branch sites

NSSA (Not So Stubby Area, RFC 3101):
  - Stub that CAN import external routes
  - Uses Type-7 LSA (translated to Type-5 at ABR)
  - Used when ASBR is inside a stub-like area

=== OSPF Router Types ===

IR  (Internal Router):   All interfaces in one area
ABR (Area Border Router): Interfaces in multiple areas; summarises
ASBR (AS Boundary Router): Redistributes external routes into OSPF
DR  (Designated Router): Elected on multi-access segments (reduces adjacencies)
BDR (Backup DR):         Takes over if DR fails
EOF
"
```

## Step 3: OSPF Packet Types and LSA Types

```bash
docker run --rm ubuntu:22.04 bash -c "
cat << 'EOF'
=== OSPF Packet Types (IP Protocol 89) ===

Type 1 - Hello
  Interval: 10s (broadcast), 30s (NBMA)
  Dead interval: 4x Hello
  Contains: Router ID, Area, Auth, Priority, DR/BDR, neighbour list
  Purpose: Discover neighbours, elect DR/BDR, maintain adjacency

Type 2 - DBD (Database Description)
  Summarises LSDB during ExStart/Exchange states
  Master/slave relationship determines who sends first
  Sequence numbers for reliable exchange

Type 3 - LSR (Link State Request)
  Requests specific LSAs missing from local LSDB
  Sent after DBD exchange to fill gaps

Type 4 - LSU (Link State Update)
  Contains one or more LSAs
  Acknowledged by LSAck

Type 5 - LSAck (Link State Acknowledgment)
  Confirms receipt of LSU

=== LSA Types ===
Type 1 - Router LSA:     Generated by every router; describes links
Type 2 - Network LSA:    Generated by DR; describes multi-access segment
Type 3 - Summary LSA:    Generated by ABR; inter-area prefixes
Type 4 - ASBR Summary:   Location of ASBR (generated by ABR)
Type 5 - AS External:    External routes (generated by ASBR)
Type 6 - Group Member:   Multicast (deprecated)
Type 7 - NSSA External:  External routes inside NSSA (converted to Type-5 at ABR)
EOF
"
```

📸 **Verified Output:**
```

Type 1 - Hello
```

> 💡 **DR/BDR Election:** On Ethernet (multi-access), OSPF elects a DR and BDR to reduce the O(n²) adjacency problem to O(n). Priority 1–255 (higher wins); if tied, highest Router ID wins. Priority 0 = never become DR.

## Step 4: OSPF Neighbour State Machine

```bash
docker run --rm ubuntu:22.04 bash -c "
cat << 'EOF'
=== OSPF Neighbour States ===

DOWN     → No Hello received from neighbour
ATTEMPT  → (NBMA only) Unicast Hello sent; no response yet
INIT     → Hello received but our Router ID not in neighbour list
2-WAY    → Bidirectional communication confirmed; DR/BDR election here
           (Non-DR/BDR routers stop here for adjacency)
EXSTART  → Master/slave negotiation begins; initial DBD exchange
EXCHANGE → Full DBD summaries exchanged; LSDB comparison
LOADING  → LSR sent; LSU received; filling LSDB gaps
FULL     → Complete adjacency; LSDBs synchronised ✓

Key: Only DR and BDR form FULL adjacency with all routers on segment.
     Other routers are in 2-WAY state with each other (DROthers).

Adjacency vs Neighbour:
  Neighbour  = Router seen in Hello packets (2-WAY+)
  Adjacency  = Full LSDB synchronisation (FULL state)
EOF
"
```

## Step 5: Write a FRRouting OSPF Configuration

```bash
docker run --rm ubuntu:22.04 bash -c "
apt-get update -qq && apt-get install -y -qq frr 2>/dev/null | tail -1

cat > /etc/frr/ospfd.conf << 'OSPFCONF'
! ============================================
! FRRouting OSPFd Configuration
! Router: core-router-1
! ============================================
hostname core-router-1
log syslog informational

router ospf
 ospf router-id 1.1.1.1
 !
 ! Area 0 — backbone
 network 10.0.0.0/30 area 0.0.0.0
 network 10.0.1.0/30 area 0.0.0.0
 network 192.168.0.0/24 area 0.0.0.0
 !
 ! Area 1 — stub (branch office)
 network 10.1.0.0/24 area 0.0.0.1
 area 0.0.0.1 stub
 !
 ! Area 2 — NSSA (remote site with external connectivity)
 network 10.2.0.0/24 area 0.0.0.2
 area 0.0.0.2 nssa
 !
 ! Timers
 timers throttle spf 50 1000 10000
 !
 ! Route summarisation at ABR
 area 0.0.0.1 range 10.1.0.0/16
 area 0.0.0.2 range 10.2.0.0/16
 !
 ! Redistribute static routes (ASBR function)
 redistribute static metric-type 2
 !
 ! Authentication for Area 0
 area 0.0.0.0 authentication message-digest
 !
 ! Passive interface (don't send OSPF on this segment)
 passive-interface eth0

! Interface-level settings
interface eth1
 ip ospf hello-interval 10
 ip ospf dead-interval 40
 ip ospf priority 255
 ip ospf message-digest-key 1 md5 s3cr3tK3y
 ip ospf cost 10

!
line vty
OSPFCONF

echo '=== OSPF config written ==='
wc -l /etc/frr/ospfd.conf
echo ''
echo '=== Show vtysh OSPF commands available ==='
cat << 'CMDS'
# Runtime verification commands (vtysh):
vtysh -c 'show ip ospf'                    # OSPF process summary
vtysh -c 'show ip ospf neighbor'           # Neighbour table
vtysh -c 'show ip ospf neighbor detail'    # Full neighbour state
vtysh -c 'show ip ospf database'           # Full LSDB
vtysh -c 'show ip ospf database router'    # Type-1 LSAs only
vtysh -c 'show ip ospf route'              # OSPF RIB
vtysh -c 'show ip route ospf'             # Installed OSPF routes
vtysh -c 'debug ospf packet all'           # Packet-level debug
CMDS
"
```

## Step 6: OSPF Authentication Modes

```bash
docker run --rm ubuntu:22.04 bash -c "
cat << 'EOF'
=== OSPF Authentication Types ===

Type 0 — Null (no authentication)
  ip ospf authentication null

Type 1 — Plaintext password
  ip ospf authentication
  ip ospf authentication-key MyPlaintextPW
  Risk: Password visible in wireshark/tcpdump!

Type 2 — MD5 HMAC (recommended)
  ip ospf authentication message-digest
  ip ospf message-digest-key 1 md5 MyMD5Key
  - Key ID (1-255) allows key rotation
  - MD5 hash included in each OSPF packet
  - Replay protection via sequence number

OSPFv3 Authentication (IPv6):
  - Uses IPsec AH/ESP (RFC 4552)
  - No built-in auth field in OSPFv3 packets
  ipv6 ospf6 authentication ipsec spi 500 md5 1234...

BIRD2 Authentication:
  protocol ospf v2 {
    area 0 {
      interface \"eth1\" {
        authentication cryptographic;
        password \"s3cr3t\" { algorithm hmac sha256; };
      };
    };
  }
EOF
"
```

📸 **Verified Output:**
```
=== OSPF Authentication Types ===

Type 0 — Null (no authentication)
  ip ospf authentication null

Type 1 — Plaintext password
  ip ospf authentication
  ip ospf authentication-key MyPlaintextPW
  Risk: Password visible in wireshark/tcpdump!

Type 2 — MD5 HMAC (recommended)
  ip ospf authentication message-digest
  ip ospf message-digest-key 1 md5 MyMD5Key
...
```

> 💡 **OSPF Cost:** OSPF uses cost = 10⁸ / bandwidth (bps). FastEthernet (100Mbps) = cost 1; T1 (1.544Mbps) ≈ 64. Set `auto-cost reference-bandwidth 100000` (100Gbps) so modern interfaces get meaningful values.

## Step 7: OSPFv3 for IPv6

```bash
docker run --rm ubuntu:22.04 bash -c "
cat << 'EOF'
=== OSPFv3 (RFC 5340) — OSPF for IPv6 ===

Key differences from OSPFv2:
  - Uses IPv6 link-local addresses for communication
  - Link-scoped flooding (multicast FF02::5/FF02::6)
  - Router ID still 32-bit (but not an IPv4 address)
  - Authentication via IPsec (no built-in auth field)
  - Separate process from OSPFv2

FRRouting OSPFv3 config (/etc/frr/ospf6d.conf):
  router ospf6
   ospf6 router-id 1.1.1.1
   !
   interface eth0 area 0.0.0.0
   interface eth1 area 0.0.0.1
   !
  interface eth0
   ipv6 ospf6 hello-interval 10
   ipv6 ospf6 dead-interval 40
   ipv6 ospf6 cost 10

BIRD2 OSPFv3 config:
  protocol ospf v3 {
    ipv6;
    area 0 {
      interface \"eth0\" { type broadcast; };
      interface \"eth1\" { type ptp; };
    };
  }

Verification:
  vtysh -c 'show ipv6 ospf6 neighbor'
  vtysh -c 'show ipv6 ospf6 database'
  vtysh -c 'show ipv6 route ospf6'
EOF
```

## Step 8: Capstone — Complete Multi-Area OSPF Design

```bash
docker run --rm ubuntu:22.04 bash -c "
apt-get update -qq && apt-get install -y -qq frr 2>/dev/null | tail -1

cat > /tmp/ospf-full.conf << 'EOF'
! ============================================
! Multi-Area OSPF Design
! Topology:
!   Area 0: Core (10.0.0.0/8)
!   Area 1: Branch A — Stub (10.1.0.0/16)
!   Area 2: Branch B — NSSA (10.2.0.0/16)
!   Area 3: DC — Totally Stubby (10.3.0.0/16)
! ============================================

router ospf
 ospf router-id 10.0.0.1
 auto-cost reference-bandwidth 100000
 !
 ! Backbone area
 network 10.0.0.0/30 area 0
 network 10.0.0.4/30 area 0
 network 10.0.0.8/30 area 0
 !
 ! Stub area — no external routes, only default
 network 10.1.0.0/24 area 1
 area 1 stub no-summary
 !
 ! NSSA — can import external routes via Type-7 LSA
 network 10.2.0.0/24 area 2
 area 2 nssa default-information-originate
 !
 ! Totally stubby — minimal LSDB
 network 10.3.0.0/24 area 3
 area 3 stub no-summary
 !
 ! Summarisation (reduces LSA flooding)
 area 1 range 10.1.0.0/16 cost 100
 area 2 range 10.2.0.0/16 cost 100
 area 3 range 10.3.0.0/16 cost 100
 !
 ! Default route redistribution
 default-information originate always metric 10 metric-type 1
EOF

echo '=== Config size ==='
wc -l /tmp/ospf-full.conf

echo ''
```

## Summary

| Topic | Key Points |
|-------|-----------|
| **Protocol Type** | Link-state IGP; IP protocol 89 |
| **Algorithm** | Dijkstra SPF on complete LSDB |
| **Area 0** | Backbone; all areas must connect here |
| **Stub** | Blocks Type-5 LSAs; ABR injects default |
| **Totally Stubby** | Blocks Type-3/4/5; minimal LSDB |
| **NSSA** | Stub that allows external routes via Type-7 |
| **Hello/Dead** | 10s/40s broadcast; 30s/120s NBMA |
| **DR/BDR** | Elected by priority then Router-ID |
| **Neighbour States** | Down→Init→2-Way→ExStart→Exchange→Loading→**Full** |
| **LSA Types** | 1=Router, 2=Network, 3=Summary, 5=External, 7=NSSA |
| **Auth** | Type 0 (none), 1 (plaintext), 2 (MD5) |
| **OSPFv3** | IPv6 variant; IPsec for auth; link-local nexthops |
| **Cost** | 10⁸/bandwidth; set ref-BW to 100Gbps |
